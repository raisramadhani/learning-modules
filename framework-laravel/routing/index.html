Dasar-Dasar Routing

Apa itu Routing?

Routing adalah mekanisme untuk menentukan bagaimana aplikasi merespons permintaan klien ke endpoint tertentu. Dalam Laravel 12, routing didefinisikan dalam file routes/web.php atau routes/api.php.

Baru di Laravel 12: Auto-discovery routes dan improved performance

Route Modern Laravel 12

// Closure route
Route::get('/', fn() => view('welcome'));
// Controller route
Route::get('/dashboard', [DashboardController::class, 'index']);

Urutan Route & Penulisan yang Benar

üéØ Prinsip Penting: Route dibaca dari ATAS ke BAWAH

Laravel akan mencocokkan route pertama yang sesuai dengan URL. Jika route sudah cocok, route di bawahnya tidak akan diperiksa lagi.

‚ùå SALAH - Route Bentrok

// Route umum di atas
Route::get('/user/{id}', function($id) {
return "User: $id";
});
// Route khusus di bawah - TIDAK AKAN TERBACA!
Route::get('/user/profile', function() {
return 'User Profile';
});

URL /user/profile akan ditangkap oleh route pertama dengan id = "profile"

‚úÖ BENAR - Urutan yang Tepat

// Route khusus di atas
Route::get('/user/profile', function() {
return 'User Profile';
});
// Route umum di bawah
Route::get('/user/{id}', function($id) {
return "User: $id";
});

Sekarang /user/profile akan ditangani dengan benar

üìã Tips Menghindari Bentrok Route:

  1. 1. Route Spesifik di Atas: Letakkan route dengan path tetap (seperti /user/profile) sebelum route dengan parameter (seperti /user/{id})
  2. 2. Gunakan Constraints: Batasi parameter dengan regex untuk menghindari konflik
  3. 3. Grouping yang Logis: Kelompokkan route berdasarkan fungsi dan tingkat spesifikasi
  4. 4. Test Urutan: Selalu test route dengan php artisan route:list
  5. 5. Dokumentasi: Beri komentar pada route yang berpotensi konflik
// Contoh urutan yang benar untuk sistem blog
// 1. Route khusus (fixed endpoints)
Route::get('/posts/create', [PostController::class, 'create']);
Route::get('/posts/search', [PostController::class, 'search']);
Route::get('/posts/popular', [PostController::class, 'popular']);
// 2. Route dengan constraint
Route::get('/posts/{id}/edit', [PostController::class, 'edit'])
->where('id', '[0-9]+');
// 3. Route umum dengan parameter
Route::get('/posts/{slug}', [PostController::class, 'show'])
->where('slug', '[a-z0-9-]+');

Route Resource - Route Gabungan Laravel

Route Resource adalah fitur Laravel yang memungkinkan kita membuat semua route CRUD (Create, Read, Update, Delete) dengan satu baris kode saja. Ini sangat menghemat waktu dan mengikuti konvensi RESTful secara otomatis.

Mengapa Menggunakan Resource Routes?

  • ‚Ä¢ Membuat 7 route sekaligus untuk operasi CRUD
  • ‚Ä¢ Mengikuti konvensi RESTful secara otomatis
  • ‚Ä¢ Menghemat waktu dan mengurangi kode berulang
  • ‚Ä¢ Konsisten dengan standar Laravel
  • ‚Ä¢ Mudah di-maintain dan dipahami developer lain

Jenis Resource Routes:

  • ‚Ä¢ Route::resource() - Web routes dengan form
  • ‚Ä¢ Route::apiResource() - API routes tanpa form
  • ‚Ä¢ Nested resources untuk relasi
  • ‚Ä¢ Partial resources (only/except)
// Web Resource (7 routes dengan create & edit forms)
Route::resource('users', UserController::class);
// API Resource (5 routes tanpa forms)
Route::apiResource('posts', PostController::class);
// Resource dengan middleware
Route::resource('admin/users', Admin\UserController::class)
->middleware(['auth', 'admin']);
// Partial resource (hanya route tertentu)
Route::resource('photos', PhotoController::class)
->only(['index', 'show']);
// Exclude route tertentu
Route::resource('comments', CommentController::class)
->except(['create', 'edit']);

Web Resource Routes (7 routes):

  • ‚Ä¢ GET /users ‚Üí index (daftar semua)
  • ‚Ä¢ GET /users/create ‚Üí create (form tambah)
  • ‚Ä¢ POST /users ‚Üí store (simpan data)
  • ‚Ä¢ GET /users/{user} ‚Üí show (detail)
  • ‚Ä¢ GET /users/{user}/edit ‚Üí edit (form edit)
  • ‚Ä¢ PUT/PATCH /users/{user} ‚Üí update (update data)
  • ‚Ä¢ DELETE /users/{user} ‚Üí destroy (hapus)

API Resource Routes (5 routes):

  • ‚Ä¢ GET /posts ‚Üí index
  • ‚Ä¢ POST /posts ‚Üí store
  • ‚Ä¢ GET /posts/{post} ‚Üí show
  • ‚Ä¢ PUT/PATCH /posts/{post} ‚Üí update
  • ‚Ä¢ DELETE /posts/{post} ‚Üí destroy
  • ‚Ä¢ Tidak ada create & edit forms

Named Routes - Penamaan Route

Named Routes adalah fitur Laravel yang memungkinkan kita memberikan nama pada route sehingga mudah direferensikan dalam aplikasi. Ini sangat berguna untuk membuat URL yang dapat berubah tanpa harus mengubah semua kode yang mereferensikannya.

‚úÖ Keuntungan Named Routes:

  • ‚Ä¢ URL bisa berubah tanpa error
  • ‚Ä¢ Code lebih mudah dibaca
  • ‚Ä¢ Mudah maintenance dan refactoring
  • ‚Ä¢ Auto-complete di IDE
  • ‚Ä¢ Type safety di Laravel 12

üéØ Kapan Menggunakan:

  • ‚Ä¢ Link navigation menu
  • ‚Ä¢ Form action URLs
  • ‚Ä¢ Redirect dalam controller
  • ‚Ä¢ Email templates
  • ‚Ä¢ API documentation

Route::get('/dashboard', [DashboardController::class, 'index'])
->name('dashboard');

// Route group dengan prefix nama
Route::name('admin.')->group(function () {

});

Route Parameters

Apa itu Route Parameters?

Route parameters adalah bagian dinamis dari URL yang memungkinkan kita menangkap nilai dari URL dan menggunakannya dalam aplikasi. Parameter sangat berguna untuk membuat route yang fleksibel dan dapat menangani berbagai input.

Mengapa Menggunakan Parameters?

  • ‚Ä¢ Membuat URL yang dinamis dan fleksibel
  • ‚Ä¢ Menghindari pembuatan route berulang
  • ‚Ä¢ Memungkinkan passing data melalui URL
  • ‚Ä¢ Membuat aplikasi lebih RESTful

Contoh Penggunaan:

  • ‚Ä¢ /user/123 - Menampilkan user dengan ID 123
  • ‚Ä¢ /product/laptop - Menampilkan produk laptop
  • ‚Ä¢ /category/electronics/item/phone - Nested parameters

Parameter Wajib (Required Parameters)

Parameter yang harus ada dalam URL agar route dapat berfungsi. Jika parameter tidak ada, route tidak akan cocok dan akan menghasilkan 404.

// Parameter tunggal
Route::get('/user/{id}', function ($id) {
return 'User ID: ' . $id;
});
// Multiple parameters
Route::get('/user/{id}/post/{postId}', function ($id, $postId) {
return "User: $id, Post: $postId";
});
// Dengan controller
Route::get('/product/{id}', [ProductController::class, 'show']);

Contoh URL yang Valid:

  • ‚úÖ /user/123 ‚Üí User ID: 123
  • ‚úÖ /user/456/post/789 ‚Üí User: 456, Post: 789
  • ‚ùå /user/ ‚Üí 404 Not Found (parameter kosong)
  • ‚ùå /user ‚Üí 404 Not Found (parameter tidak ada)

Parameter Opsional (Optional Parameters)

Parameter yang boleh ada atau tidak dalam URL. Ditandai dengan tanda tanya ? setelah nama parameter.

Kapan Menggunakan?

  • ‚Ä¢ Ketika ingin satu route menangani multiple case
  • ‚Ä¢ Untuk pagination atau filtering
  • ‚Ä¢ Ketika parameter memiliki nilai default
  • ‚Ä¢ Untuk membuat API yang fleksibel

Perbedaan dengan Required:

  • ‚Ä¢ Required: {id} - Harus ada
  • ‚Ä¢ Optional: {id?} - Boleh kosong
  • ‚Ä¢ Optional harus punya default value
  • ‚Ä¢ Optional biasanya di akhir URL
// Parameter opsional sederhana
Route::get('/posts/{id?}', function ($id = null) {
if ($id) {
return 'Post ID: ' . $id;
}
return 'Semua posts';
});
// Dengan default value
Route::get('/products/{category?}', function ($category = 'all') {
return "Category: $category";
});
// Multiple optional parameters
Route::get('/search/{query?}/{page?}', function ($query = '', $page = 1) {
return "Query: $query, Page: $page";
});

Contoh URL yang Valid:

  • ‚úÖ /posts ‚Üí Semua posts
  • ‚úÖ /posts/123 ‚Üí Post ID: 123
  • ‚úÖ /products ‚Üí Category: all
  • ‚úÖ /products/electronics ‚Üí Category: electronics
  • ‚úÖ /search ‚Üí Query: , Page: 1
  • ‚úÖ /search/laptop/2 ‚Üí Query: laptop, Page: 2

Parameter Constraints (Validasi Parameter)

Laravel 12 memungkinkan kita membatasi format parameter menggunakan regular expressions.

// Hanya angka
Route::get('/user/{id}', [UserController::class, 'show'])
->where('id', '[0-9]+');
// Hanya huruf
Route::get('/category/{name}', [CategoryController::class, 'show'])
->where('name', '[A-Za-z]+');
// Multiple constraints
Route::get('/user/{id}/post/{slug}', [PostController::class, 'show'])
->where(['id' => '[0-9]+', 'slug' => '[a-z-]+']);

HTTP Methods

Apa itu HTTP Methods?

HTTP (HyperText Transfer Protocol) Methods adalah cara untuk menentukan jenis operasi yang ingin dilakukan pada resource tertentu. Setiap method memiliki tujuan yang berbeda dalam arsitektur RESTful.

HTTP (HyperText Transfer Protocol):

Protokol komunikasi yang digunakan untuk mentransfer data antara client (browser) dan server di World Wide Web.

RESTful (Representational State Transfer):

Arsitektur web service yang menggunakan HTTP methods untuk operasi CRUD (Create, Read, Update, Delete) pada resources.

Prinsip RESTful:

  • ‚Ä¢ GET - Mengambil/membaca data (Read)
  • ‚Ä¢ POST - Membuat data baru (Create)
  • ‚Ä¢ PUT - Memperbarui seluruh data (Update/Replace)
  • ‚Ä¢ PATCH - Memperbarui sebagian data (Partial Update)
  • ‚Ä¢ DELETE - Menghapus data (Delete)
  • ‚Ä¢ OPTIONS - Mengecek method yang tersedia

GET

Mengambil/membaca data dari server tanpa mengubah apapun

Route::get('/users', [UserController::class, 'index']);
Route::get('/users/{id}', [UserController::class, 'show']);
Contoh Studi Kasus GET:
  • ‚Ä¢ E-commerce: Menampilkan daftar produk, detail produk
  • ‚Ä¢ Blog: Menampilkan artikel, kategori, komentar
  • ‚Ä¢ Social Media: Timeline, profil user, daftar followers
  • ‚Ä¢ Dashboard: Statistik, laporan, grafik
  • ‚Ä¢ API: Mengambil data JSON untuk mobile app

POST

Membuat data baru atau mengirim data ke server

Route::post('/users', [UserController::class, 'store']);
Route::post('/login', [AuthController::class, 'login']);
Contoh Studi Kasus POST:
  • ‚Ä¢ Registrasi: Membuat akun user baru
  • ‚Ä¢ Login: Autentikasi user
  • ‚Ä¢ Form Contact: Mengirim pesan/feedback
  • ‚Ä¢ Upload File: Mengunggah gambar/dokumen
  • ‚Ä¢ Checkout: Membuat order baru di e-commerce
  • ‚Ä¢ Comment: Menambah komentar di blog/forum

PUT

Memperbarui seluruh data atau mengganti resource

Route::put('/users/{id}', [UserController::class, 'update']);
Route::put('/profile', [ProfileController::class, 'update']);
Contoh Studi Kasus PUT:
  • ‚Ä¢ Update Profile: Mengganti semua data profil user
  • ‚Ä¢ Edit Article: Mengganti seluruh konten artikel
  • ‚Ä¢ Replace Settings: Mengganti semua pengaturan aplikasi
  • ‚Ä¢ Update Product: Mengganti semua info produk
  • ‚Ä¢ API Resource: Replace entire JSON object

PATCH

Memperbarui sebagian data saja

Route::patch('/users/{id}', [UserController::class, 'updatePartial']);
Route::patch('/posts/{id}/status', [PostController::class, 'updateStatus']);
Contoh Studi Kasus PATCH:
  • ‚Ä¢ Change Password: Hanya update password user
  • ‚Ä¢ Update Status: Mengubah status order (pending ‚Üí paid)
  • ‚Ä¢ Toggle Feature: Enable/disable notifikasi
  • ‚Ä¢ Partial Profile: Update hanya nama atau email
  • ‚Ä¢ Like/Unlike: Toggle like pada post
  • ‚Ä¢ Increment Counter: Update view count artikel

DELETE

Menghapus data dari server

Route::delete('/users/{id}', [UserController::class, 'destroy']);
Route::delete('/posts/{id}', [PostController::class, 'destroy']);
Contoh Studi Kasus DELETE:
  • ‚Ä¢ Delete Account: Menghapus akun user
  • ‚Ä¢ Remove Post: Menghapus artikel/post
  • ‚Ä¢ Delete File: Menghapus file upload
  • ‚Ä¢ Cancel Order: Membatalkan pesanan
  • ‚Ä¢ Remove Comment: Menghapus komentar
  • ‚Ä¢ Logout: Menghapus session/token

OPTIONS

Mengecek method yang tersedia untuk resource

Route::options('/api/users', [ApiController::class, 'options']);
Route::match(['get', 'post'], '/contact', [ContactController::class, 'handle']);
Contoh Studi Kasus OPTIONS:
  • ‚Ä¢ CORS Preflight: Browser check sebelum AJAX request
  • ‚Ä¢ API Discovery: Cek method yang tersedia di endpoint
  • ‚Ä¢ Security Check: Validasi akses sebelum request utama
  • ‚Ä¢ Mobile App: Cek API capabilities
  • ‚Ä¢ Third-party Integration: Validasi endpoint

HTTP Response dalam Laravel 12

Laravel 12 menyediakan berbagai cara untuk mengembalikan response yang sesuai dengan HTTP method.

Response Types:

// JSON Response (API)
return response()->json([
'data' => $users,
'status' => 'success'
]);
// View Response
return view('users.index', compact('users'));
// Redirect Response
return redirect()->route('users.index')
->with('success', 'User created!');

Status Codes:

// Success responses
return response()->json($data, 200); // OK
return response()->json($data, 201); // Created
return response()->json([], 204); // No Content
// Error responses
return response()->json(['error' => 'Not found'], 404);

Best Practices Response:

  • ‚Ä¢ GET ‚Üí 200 (OK) dengan data, 404 jika tidak ditemukan
  • ‚Ä¢ POST ‚Üí 201 (Created) dengan data baru, 422 jika validasi gagal
  • ‚Ä¢ PUT/PATCH ‚Üí 200 (OK) dengan data updated, 404 jika tidak ditemukan
  • ‚Ä¢ DELETE ‚Üí 204 (No Content) atau 200 dengan pesan konfirmasi

Middleware & Authentication

Route dengan Auth Middleware

Melindungi route agar hanya bisa diakses oleh user yang sudah login.

// Single route dengan auth
Route::get('/profile', [ProfileController::class, 'show'])
->middleware('auth');
// Multiple middleware
Route::get('/admin', [AdminController::class, 'index'])
->middleware(['auth', 'verified', 'admin']);

Middleware Bawaan Laravel 12:

  • auth - Memastikan user sudah login
  • verified - Memastikan email sudah diverifikasi
  • guest - Hanya untuk user yang belum login
  • throttle - Rate limiting

Route Groups dengan Middleware

Mengelompokkan route dengan middleware yang sama.

// Group dengan auth middleware
Route::middleware('auth')->group(function () {
Route::get('/dashboard', [DashboardController::class, 'index']);
Route::get('/profile', [ProfileController::class, 'show']);
Route::post('/profile', [ProfileController::class, 'update']);
});

Custom Middleware

Membuat middleware kustom untuk kebutuhan spesifik.

// Membuat middleware
php artisan make:middleware CheckAge
// Menggunakan custom middleware
Route::get('/adult-content', [ContentController::class, 'adult'])
->middleware('check.age:18');

Security & Rate Limiting

Rate Limiting

Membatasi jumlah request per waktu untuk mencegah abuse.

// Rate limit 60 requests per menit
Route::get('/api/data', [ApiController::class, 'data'])
->middleware('throttle:60,1');
// Rate limit berbeda untuk user login
Route::post('/api/upload', [UploadController::class, 'store'])
->middleware('throttle:uploads');

Rate Limiting di Laravel 12:

  • ‚Ä¢ throttle:60,1 - 60 requests per menit
  • ‚Ä¢ throttle:api - Menggunakan konfigurasi default API
  • ‚Ä¢ throttle:uploads - Named rate limiter kustom

CSRF Protection

Perlindungan terhadap Cross-Site Request Forgery attacks.

// Route dengan CSRF protection (default untuk web routes)
Route::post('/contact', [ContactController::class, 'store']);
// Exclude dari CSRF (hati-hati!)
Route::post('/webhook', [WebhookController::class, 'handle'])
->withoutMiddleware([VerifyCsrfToken::class]);

CORS Configuration

Mengatur Cross-Origin Resource Sharing untuk API.

// API routes dengan CORS
Route::prefix('api')->middleware(['cors', 'throttle:api'])->group(function () {
Route::get('/users', [UserApiController::class, 'index']);
Route::post('/users', [UserApiController::class, 'store']);
});

Latihan Soal Laravel Routing

MUDAH

Soal 1: Route Dasar

Buatlah route sederhana yang menampilkan halaman "Contact" ketika user mengakses URL /contact. Gunakan closure (function) untuk mengembalikan teks "Halaman Kontak".

Solusi:

Route::get('/contact', function () {
return 'Halaman Kontak';
});

Penjelasan: Route dasar menggunakan Route::get() dengan closure function untuk mengembalikan response sederhana berupa string.

MUDAH-SEDANG

Soal 2: Route dengan Controller

Buatlah route yang menampilkan halaman "About Us" ketika user mengakses URL /about. Gunakan syntax array-based modern dan arahkan ke PageController method about.

Solusi:

Route::get('/about', [PageController::class, 'about']);

Penjelasan: Menggunakan syntax array-based [PageController::class, 'about'] yang merupakan standar Laravel 12 untuk mengarahkan ke controller method.

SEDANG

Soal 3: Route dengan Parameter dan Validasi

Buatlah route yang menampilkan detail produk berdasarkan ID. URL: /product/{id} dimana ID harus berupa angka saja (gunakan constraint). Arahkan ke ProductController method show.

Solusi:

Route::get('/product/{id}', [ProductController::class, 'show'])
->where('id', '[0-9]+');

Penjelasan: Menggunakan where('id', '[0-9]+') untuk membatasi parameter ID hanya menerima angka. Regex [0-9]+ berarti satu atau lebih digit angka.

SEDANG-SULIT

Soal 4: Resource Routes dengan Pembatasan

Buatlah resource route untuk mengelola data "posts" yang hanya memiliki operasi: index, show, store, dan destroy (tanpa create dan edit forms). Gunakan PostController dan tambahkan middleware auth untuk semua route.

Solusi:

Route::resource('posts', PostController::class)
->only(['index', 'show', 'store', 'destroy'])
->middleware('auth');

Penjelasan: Menggunakan Route::resource() dengan only() untuk membatasi route tertentu, dan middleware('auth') untuk proteksi autentikasi.

Soal 5: Route Group Kompleks dengan Multiple Middleware

Buatlah route group untuk admin panel dengan konfigurasi berikut:
‚Ä¢ Prefix: admin
‚Ä¢ Middleware: auth, verified, dan admin
‚Ä¢ Rate limiting: maksimal 100 request per menit
‚Ä¢ Berisi 3 route: dashboard (GET), users (GET), dan settings (POST)
‚Ä¢ Semua menggunakan AdminController dengan method yang sesuai

Solusi:

Route::prefix('admin')
->middleware(['auth', 'verified', 'admin', 'throttle:100,1'])
->group(function () {
Route::get('/dashboard', [AdminController::class, 'dashboard']);
Route::get('/users', [AdminController::class, 'users']);
Route::post('/settings', [AdminController::class, 'settings']);
});

Penjelasan: Route group kompleks dengan multiple middleware termasuk rate limiting throttle:100,1, prefix untuk URL, dan berbagai HTTP methods dalam satu group.
