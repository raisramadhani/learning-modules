Week 10: Generics dan Collections

Memahami Collections Framework dan Generics untuk mengelola kumpulan data dengan type-safe

Apa itu Generics?

Generics adalah fitur yang diperkenalkan pada JDK 5.0 untuk memberikan keamanan tipe data (type safety) dan mengurangi bug pada saat kompilasi, bukan runtime.

Tujuan Generics: Memungkinkan kelas, interface, atau method beroperasi pada tipe data yang ditentukan saat deklarasi, sehingga menghindari casting manual dan error saat runtime.

Tanpa Generics vs Dengan Generics:

Tanpa Generics

ArrayList list = new ArrayList();
list.add("Hello");
list.add(123);

String str = (String) list.get(0);
String num = (String) list.get(1);
// Runtime error!

Error terjadi saat runtime, casting manual diperlukan

Dengan Generics

ArrayList<String> list = new ArrayList<>();
list.add("Hello");
list.add(123);
// Compile error!

String str = list.get(0);
// Tidak perlu casting

Error terdeteksi saat kompilasi, type safety terjamin

Diamond Operator (Java 7+):

// Sebelum Java 7
ArrayList<String> list = new ArrayList<String>();

// Java 7+ dengan Diamond Operator
ArrayList<String> list = new ArrayList<>();
// Compiler otomatis mendeteksi tipe data

Java Collections Framework

Java Collections Framework adalah arsitektur terpadu untuk menyimpan dan memanipulasi kelompok objek. Framework ini menyediakan interface dan implementasi untuk berbagai struktur data.

Hirarki Collections:

Collection (Interface)
├── List (Interface)
│   ├── ArrayList
│   ├── LinkedList
│   └── Vector
├── Set (Interface)
│   ├── HashSet
│   └── TreeSet
└── Queue (Interface)
    ├── LinkedList
    └── PriorityQueue

Map (Interface) - Terpisah dari Collection
├── HashMap
├── TreeMap
└── LinkedHashMap

List Interface

List adalah koleksi terurut yang mengizinkan elemen duplikat. Elemen dapat diakses menggunakan index.

ArrayList

  • Struktur Data: Array dinamis
  • Akses Elemen: Cepat (O(1))
  • Penyisipan/Penghapusan: Lambat di tengah (O(n))
  • Use Case: Akses random yang sering, jarang insert/delete di tengah
ArrayList<String> fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Cherry");
fruits.add(1, "Mango");

String first = fruits.get(0);
fruits.remove("Banana");
int size = fruits.size();

for (String fruit : fruits) {
    System.out.println(fruit);
}

LinkedList

  • Struktur Data: Doubly-linked list
  • Akses Elemen: Lambat (O(n))
  • Penyisipan/Penghapusan: Cepat di awal/akhir (O(1))
  • Use Case: Sering insert/delete, jarang akses random
LinkedList<Integer> numbers = new LinkedList<>();
numbers.add(10);
numbers.addFirst(5);
numbers.addLast(15);

int first = numbers.getFirst();
int last = numbers.getLast();
numbers.removeFirst();

Set Interface

Set adalah koleksi yang tidak mengizinkan elemen duplikat. Set tidak menjaga urutan elemen (kecuali TreeSet).

HashSet

Urutan: Tidak terurut

Performance: O(1) untuk add, remove, contains

Null: Mengizinkan 1 null

Use Case: Cek keunikan data cepat

TreeSet

Urutan: Terurut (sorted)

Performance: O(log n) untuk operasi

Null: Tidak mengizinkan null

Use Case: Data unik dan terurut

// HashSet
HashSet<String> set = new HashSet<>();
set.add("Java");
set.add("Python");
set.add("Java");
System.out.println(set.size());

boolean hasJava = set.contains("Java");
set.remove("Python");

// TreeSet
TreeSet<Integer> numbers = new TreeSet<>();
numbers.add(5);
numbers.add(2);
numbers.add(8);
numbers.add(1);
System.out.println(numbers);

Queue Interface

Queue adalah koleksi yang mengikuti prinsip FIFO (First-In-First-Out) atau prioritas. Digunakan untuk mengelola antrean proses.

// LinkedList sebagai Queue
Queue<String> queue = new LinkedList<>();
queue.offer("First");
queue.offer("Second");
queue.offer("Third");

String head = queue.peek();
String removed = queue.poll();

// PriorityQueue
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(5);
pq.offer(1);
pq.offer(3);
System.out.println(pq.poll());
Method Queue:
  • offer() - Menambah elemen ke queue
  • poll() - Mengambil dan menghapus head
  • peek() - Melihat head tanpa menghapus

Map Interface

Map adalah koleksi pasangan kunci-nilai (key-value), di mana setiap kunci harus unik. Map tidak termasuk dalam Collection interface.

HashMap

Tidak terurut

Performance: O(1)

Null key: 1 null key

Thread-safe: Tidak

TreeMap

Terurut by key

Performance: O(log n)

Null key: Tidak

Thread-safe: Tidak

LinkedHashMap

Terurut by insertion

Performance: O(1)

Null key: 1 null key

Thread-safe: Tidak

// HashMap
HashMap<String, Integer> scores = new HashMap<>();
scores.put("Alice", 90);
scores.put("Bob", 85);
scores.put("Charlie", 95);

int aliceScore = scores.get("Alice");
boolean hasBob = scores.containsKey("Bob");
scores.remove("Charlie");

// Iterasi Map
for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

// Get keys and values
Set<String> keys = scores.keySet();
Collection<Integer> values = scores.values();

Collections Utility Class

Kelas Collections menyediakan method statis untuk operasi umum pada koleksi seperti pengurutan, pencarian, dan manipulasi.

Method Umum Collections:

List<Integer> numbers = new ArrayList<>();
numbers.add(5);
numbers.add(2);
numbers.add(8);
numbers.add(1);
numbers.add(9);

// Sort
Collections.sort(numbers);
System.out.println(numbers);

// Reverse
Collections.reverse(numbers);

// Shuffle
Collections.shuffle(numbers);

// Max dan Min
int max = Collections.max(numbers);
int min = Collections.min(numbers);

// Binary Search
int index = Collections.binarySearch(numbers, 5);

// Frequency
int count = Collections.frequency(numbers, 5);

// Fill
Collections.fill(numbers, 0);

// Copy
List<Integer> copy = new ArrayList<>(numbers.size());
Collections.copy(copy, numbers);
