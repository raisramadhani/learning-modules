Week 1: Class & Object

Memahami dasar-dasar OOP: Class, Object, Constructor, dan Memory Management

Pemrograman Berorientasi Objek (OOP)

OOP adalah paradigma pemrograman yang berbasis pada konsep "objek" yang merepresentasikan entitas dunia nyata.

Bayangkan dunia nyata: mobil, manusia, hewan - semuanya adalah objek dengan karakteristik (atribut) dan kemampuan (metode).

Apa itu Class?

Class adalah blueprint atau template untuk membuat objek. Class berisi:

  • Atribut (Variabel): Karakteristik dari objek
  • Metode (Fungsi): Perilaku atau aksi yang bisa dilakukan objek
// Contoh Class Mobil
public class Mobil {
    // Atribut
    String warna;
    String merk;
    int kecepatan;
    
    // Metode
    void nyalakan() {
        System.out.println("Mobil dinyalakan");
    }
    
    void berjalan() {
        System.out.println("Mobil berjalan");
    }
}

Apa itu Object?

Object adalah instance (wujud nyata) dari sebuah class. Jika class adalah cetakan kue, object adalah kue yang sudah jadi!

// Membuat Object dari Class Mobil
Mobil mobil1 = new Mobil();
mobil1.warna = "Merah";
mobil1.merk = "Toyota";
mobil1.kecepatan = 0;

Mobil mobil2 = new Mobil();
mobil2.warna = "Biru";
mobil2.merk = "Honda";
mobil2.kecepatan = 0;

// Memanggil metode
mobil1.nyalakan();  // Output: Mobil dinyalakan
mobil1.berjalan();  // Output: Mobil berjalan

Constructor

Constructor adalah metode khusus yang dipanggil otomatis saat objek dibuat. Fungsinya untuk menginisialisasi nilai awal objek.

Karakteristik Constructor:

  • Nama sama dengan nama class
  • Tidak memiliki tipe return (bahkan void)
  • Dipanggil otomatis dengan keyword new
public class Mobil {
    String warna;
    String merk;
    int kecepatan;
    
    // Constructor
    public Mobil(String warna, String merk) {
        this.warna = warna;
        this.merk = merk;
        this.kecepatan = 0;
    }
}

// Menggunakan Constructor
Mobil mobil1 = new Mobil("Merah", "Toyota");
// Sekarang mobil1 langsung punya warna dan merk!

Memory Allocation di Java

Stack Memory

  • Menyimpan referensi ke objek dan variabel lokal
  • Ukuran lebih kecil dan akses lebih cepat
  • Otomatis dibersihkan saat metode selesai

Heap Memory

  • Menyimpan objek aktual yang dibuat dengan new
  • Ukuran lebih besar, akses lebih lambat
  • Dikelola oleh Garbage Collector

Garbage Collection

  • Java mengelola memori secara otomatis
  • Objek yang tidak lagi direferensi akan dihapus otomatis
  • Programmer tidak perlu manual delete objek seperti di C++
// Contoh Memory Allocation
Mobil mobil1 = new Mobil("Merah", "Toyota");
// Stack: menyimpan variabel 'mobil1' (referensi)
// Heap: menyimpan objek Mobil dengan data "Merah", "Toyota"

mobil1 = null; // Referensi dihapus
// Objek di heap sekarang tidak punya referensi
// Garbage Collector akan membersihkannya

Pertanyaan: Apa perbedaan utama antara Class dan Object?

A. Tidak ada perbedaan, keduanya sama
B. Class adalah template/blueprint, Object adalah instance dari class
C. Class lebih besar dari Object
D. Object hanya bisa dibuat sekali dari Class

Ringkasan Week 1

  • OOP adalah paradigma pemrograman berbasis objek
  • Class adalah blueprint untuk membuat objek
  • Object adalah instance dari class
  • Constructor menginisialisasi objek saat dibuat
  • Stack menyimpan referensi, Heap menyimpan objek
  • Garbage Collection membersihkan objek otomatis
