Week 7: Abstraction

Memahami abstraksi, abstract class, interface, dan kapan menggunakannya

Apa itu Abstraction?

Abstraksi adalah konsep menyembunyikan detail implementasi yang kompleks dan hanya menampilkan fitur esensial dari objek.

ğŸ’¡ Analogi: Saat Anda mengendarai mobil, Anda hanya perlu tahu cara menggunakan setir, pedal gas, dan rem. Anda tidak perlu tahu detail bagaimana mesin bekerja di dalam. Itu adalah abstraksi!

Tujuan Abstraksi:

  • Simplifikasi - Menyederhanakan penggunaan objek
  • Hide Complexity - Menyembunyikan detail implementasi
  • Maintainability - Mudah diubah tanpa mempengaruhi kode lain
  • Contract - Memisahkan "apa" dari "bagaimana"

Abstract Class

Abstract Class adalah class yang tidak bisa diinstansiasi langsung dan bisa berisi method abstrak maupun konkret.

Karakteristik Abstract Class:

  • âŒ Tidak bisa di-new (diinstansiasi)
  • âœ“ Bisa berisi method abstrak (tanpa body)
  • âœ“ Bisa berisi method konkret (dengan body)
  • âœ“ Bisa berisi constructor, atribut
  • âœ“ Digunakan dengan extends
  • âš ï¸ Subclass harus implementasi semua method abstrak
// Abstract Class
abstract class Animal {
    String name;
    
    // Constructor
    public Animal(String name) {
        this.name = name;
    }
    
    // Abstract method - tidak punya implementasi
    abstract void makeSound();
    
    // Concrete method - punya implementasi
    void sleep() {
        System.out.println(name + " is sleeping");
    }
}

// Subclass harus implementasi method abstrak
class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    void makeSound() {
        System.out.println(name + " says: Woof!");
    }
}

// Penggunaan
// Animal animal = new Animal("Generic"); // âŒ ERROR! Cannot instantiate
Dog dog = new Dog("Buddy");  // âœ“ OK

dog.sleep();      // Output: Buddy is sleeping

Interface

Interface adalah contract/kontrak yang mendefinisikan method apa yang harus diimplementasikan tanpa menentukan bagaimana implementasinya.

Karakteristik Interface:

  • Semua method adalah public abstract (default)
  • Semua atribut adalah public static final (konstanta)
  • Mendukung multiple inheritance (class bisa implements banyak interface)
  • Java 8+: Bisa punya default dan static method
  • âœ“ Digunakan dengan implements
// Interface
interface Flyable {
    // Abstract method (implicitly public abstract)
    void fly();
    void land();
    
    // Default method (Java 8+)
    default void displayFlying() {
        System.out.println("This can fly!");
    }
}

interface Swimmable {
    void swim();
}

// Class implements multiple interfaces
class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("Duck is flying");
    }
    
    @Override
    public void land() {
        System.out.println("Duck is landing");
    }
    
    @Override
    public void swim() {
        System.out.println("Duck is swimming");
    }
}

// Penggunaan
Duck duck = new Duck();
duck.fly();    // Duck is flying
duck.swim();   // Duck is swimming
duck.land();   // Duck is landing
duck.displayFlying(); // This can fly!

Kapan Menggunakan?

âœ… Gunakan Abstract Class Jika:

  • Ingin berbagi kode (method konkret) dengan subclass
  • Punya atribut instance yang perlu diwariskan
  • Perlu access modifier selain public
  • Subclass punya hubungan "is-a" yang erat
abstract class Shape {
    String color;  // Shared attribute
    
    abstract double getArea();  // Must implement
    
    void displayColor() {  // Shared behavior
        System.out.println("Color: " + color);
    }
}

âœ… Gunakan Interface Jika:

  • Ingin mendefinisikan kontrak/capability
  • Perlu multiple inheritance
  • Class tidak punya hubungan hierarki yang jelas
  • Fokus pada "apa yang bisa dilakukan" bukan "apa itu"
interface Printable {
    void print();  // Contract
}

// Any class can be Printable, tidak harus related
class Document implements Printable { ... }
class Image implements Printable { ... }
class Report implements Printable { ... }

Pertanyaan: Apa perbedaan utama antara Abstract Class dan Interface?

A. Tidak ada perbedaan
B. Abstract class bisa punya method konkret dan constructor, interface tidak. Interface support multiple inheritance
C. Abstract class lebih cepat dari interface
D. Interface hanya untuk Java 8+

Ringkasan Week 7

  • Abstraksi: Menyembunyikan detail, tampilkan fitur esensial
  • Abstract Class: Tidak bisa diinstansiasi, bisa punya method abstrak & konkret
  • Interface: Kontrak method, semua method abstract (default)
  • Multiple Inheritance: Hanya bisa via interface
  • Abstract vs Interface: Abstract untuk hubungan erat, Interface untuk capability
  • Java 8+: Interface bisa punya default & static method
