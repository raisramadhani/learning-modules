Week 12: Java I/O

Memahami mekanisme Input/Output untuk komunikasi data eksternal

Konsep Dasar I/O

I/O (Input/Output) adalah komunikasi antara aplikasi Java dan dunia luar, seperti file, keyboard, network, atau perangkat lain. Semua I/O berbasis konsep Stream.

Stream: Abstraksi yang merepresentasikan aliran data sekuensial. Stream bisa berupa input (membaca data) atau output (menulis data).

Input Stream

Membaca data dari sumber ke program

Contoh: FileInputStream, BufferedReader

Output Stream

Menulis data dari program ke tujuan

Contoh: FileOutputStream, BufferedWriter

Jenis Stream

Byte Stream

Fungsi: Menangani data biner mentah (8-bit bytes)

Kegunaan: Untuk gambar, audio, video, atau binary files

Kelas: FileInputStream, FileOutputStream

// Membaca byte dari file
FileInputStream fis = new FileInputStream("image.jpg");
int data = fis.read();
fis.close();

// Menulis byte ke file
FileOutputStream fos = new FileOutputStream("output.dat");
fos.write(65);
fos.close();

Character Stream

Fungsi: Menangani data teks (16-bit Unicode characters)

Kegunaan: Untuk membaca/menulis file teks

Kelas: FileReader, FileWriter, BufferedReader, BufferedWriter

// Membaca karakter dari file
FileReader fr = new FileReader("data.txt");
int ch = fr.read();
fr.close();

// Menulis karakter ke file
FileWriter fw = new FileWriter("output.txt");
fw.write("Hello World");
fw.close();
Best Practice: Gunakan Buffered Stream (BufferedReader, BufferedWriter) untuk performa lebih baik karena mengurangi akses disk.

Pengelolaan File

Kelas File digunakan untuk operasi file dan direktori seperti membuat, menghapus, dan memeriksa properti.

import java.io.File;

File file = new File("data.txt");

// Cek keberadaan file
if (file.exists()) {
    System.out.println("File ditemukan");
}

// Properti file
System.out.println("Nama: " + file.getName());
System.out.println("Path: " + file.getPath());
System.out.println("Ukuran: " + file.length() + " bytes");
System.out.println("Bisa dibaca: " + file.canRead());
System.out.println("Bisa ditulis: " + file.canWrite());

// Operasi file
file.createNewFile();
file.delete();

// Direktori
File dir = new File("myFolder");
dir.mkdir();
String[] files = dir.list();

Serialisasi

Serialisasi adalah proses mengubah objek menjadi byte stream untuk penyimpanan atau transmisi. Deserialisasi adalah kebalikannya.

Syarat: Kelas harus implement interface Serializable 
import java.io.*;

// Kelas yang bisa diserialisasi
class Student implements Serializable {
    private String name;
    private int age;
    private transient String password;
    
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// Serialisasi (simpan objek)
Student student = new Student("Budi", 20);
ObjectOutputStream oos = new ObjectOutputStream(
    new FileOutputStream("student.ser")
);
oos.writeObject(student);
oos.close();

// Deserialisasi (baca objek)
ObjectInputStream ois = new ObjectInputStream(
    new FileInputStream("student.ser")
);
Student loadedStudent = (Student) ois.readObject();
ois.close();
Keyword transient: Field yang ditandai transient tidak akan diserialisasi (berguna untuk data sensitif seperti password).

Random Access File

RandomAccessFile memungkinkan membaca dan menulis data pada posisi mana pun dalam file, tidak harus sekuensial.

import java.io.RandomAccessFile;

RandomAccessFile raf = new RandomAccessFile("data.dat", "rw");

// Menulis data
raf.writeInt(100);
raf.writeDouble(3.14);
raf.writeUTF("Hello");

// Pindah ke posisi tertentu
raf.seek(0);

// Membaca dari posisi awal
int num = raf.readInt();
double pi = raf.readDouble();
String text = raf.readUTF();

// Cek posisi saat ini
long position = raf.getFilePointer();

raf.close();
Mode:
  • "r" - Read only
  • "rw" - Read and Write
  • "rws" - Read, Write, dan sync metadata
  • "rwd" - Read, Write, dan sync konten

Pertanyaan 1: Apa perbedaan Byte Stream dan Character Stream?

A. Tidak ada perbedaan
B. Byte Stream untuk data biner, Character Stream untuk teks
C. Byte Stream lebih cepat dari Character Stream
D. Character Stream hanya untuk bahasa Inggris

Pertanyaan 2: Apa syarat kelas agar bisa diserialisasi?

A. Extends dari Object
B. Implement interface Serializable
C. Memiliki constructor default
D. Semua field harus public

Pertanyaan 3: Untuk apa keyword transient digunakan?

A. Membuat field menjadi temporary
B. Menandai field agar tidak diserialisasi
C. Meningkatkan performa serialisasi
D. Membuat field bisa diakses dari luar

Ringkasan Week 12

  • Stream: Abstraksi aliran data untuk I/O operations
  • Byte Stream: FileInputStream/FileOutputStream untuk data biner
  • Character Stream: FileReader/FileWriter untuk data teks
  • Buffered Stream: BufferedReader/BufferedWriter untuk performa lebih baik
  • File Class: Operasi file seperti exists(), delete(), createNewFile()
  • Serialisasi: Mengubah objek ke byte stream (implements Serializable)
  • Transient: Field yang tidak diserialisasi
  • RandomAccessFile: Akses file non-sekuensial dengan seek()
