Week 3: Penguatan & Encapsulation

Mendalami scope variabel dan prinsip enkapsulasi dalam OOP

Penguatan: Scope Variabel

Mari kita pahami lebih dalam tentang Local, Instance, dan Static Variable.

Visualisasi: Scope Variabel

Contoh Lengkap Scope Variabel

public class Mahasiswa {
    // Static Variable - Milik Class
    static int jumlahMahasiswa = 0;
    
    // Instance Variables - Milik setiap objek
    String nama;
    int umur;
    
    // Constructor
    public Mahasiswa(String nama, int umur) {
        this.nama = nama;      // Instance variable
        this.umur = umur;      // Instance variable
        jumlahMahasiswa++;     // Static variable
    }
    
    public void tampilkanInfo() {
        // Local Variable - Hanya ada di method ini
        String pesan = "Data Mahasiswa";
        
        System.out.println(pesan);           // ‚úì Bisa diakses
        System.out.println("Nama: " + nama); // ‚úì Bisa diakses
        System.out.println("Total: " + jumlahMahasiswa); // ‚úì Bisa diakses
    }
    
    public void methodLain() {
        // System.out.println(pesan); // ‚ùå ERROR! pesan tidak bisa diakses
        System.out.println(nama);     // ‚úì Instance variable bisa diakses
    }
}

Encapsulation (Enkapsulasi)

Enkapsulasi adalah prinsip OOP untuk menyembunyikan detail internal objek dan hanya mengekspos antarmuka publik yang diperlukan.

Tujuan Enkapsulasi:

  • Keamanan Data - Melindungi data dari akses tidak sah
  • Kontrol Akses - Mengatur bagaimana data diakses dan dimodifikasi
  • Fleksibilitas - Mudah mengubah implementasi tanpa mempengaruhi kode lain
  • Modularitas - Memisahkan interface dari implementasi

Getter & Setter Methods

Enkapsulasi diimplementasikan dengan membuat atribut private dan menyediakan method public untuk mengakses/mengubahnya. 

public class BankAccount {
    // Atribut private - Tidak bisa diakses langsung dari luar
    private String accountNumber;
    private double balance;
    
    // Constructor
    public BankAccount(String accountNumber, double initialBalance) {
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }
    
    // Getter - Mengambil nilai
    public double getBalance() {
        return balance;
    }
    
    public String getAccountNumber() {
        return accountNumber;
    }
    
    // Setter dengan validasi - Mengubah nilai
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposit berhasil!");
        } else {
            System.out.println("Jumlah harus positif!");
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Penarikan berhasil!");
        } else {
            System.out.println("Saldo tidak cukup!");
        }
    }
}

// Penggunaan
BankAccount akun = new BankAccount("1234567890", 1000000);
// akun.balance = -5000;  // ‚ùå ERROR! balance is private
akun.deposit(500000);      // ‚úì Menggunakan method public
System.out.println(akun.getBalance());  // ‚úì Mengambil nilai dengan getter

Ringkasan Week 3

  • Local Variable: Dalam method, harus diinisialisasi, tidak punya default
  • Instance Variable: Dalam class, punya nilai default, tiap objek punya copy
  • Static Variable: Milik class, shared oleh semua objek
  • Enkapsulasi: Menyembunyikan detail internal dengan private
  • Access Modifiers: private, default, protected, public
  • Getter/Setter: Method untuk akses/modifikasi data yang ter-enkapsulasi
