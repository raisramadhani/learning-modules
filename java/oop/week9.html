Week 9: Operasi String

Memahami karakteristik String, String Pool, dan manipulasi teks dalam Java

Apa itu String?

String dalam Java adalah urutan karakter yang merupakan objek dari paket java.lang. String adalah salah satu tipe data yang paling sering digunakan dalam pemrograman.

Karakteristik Utama String: String bersifat immutable (tidak dapat diubah). Setiap modifikasi pada string akan menghasilkan objek string baru, bukan mengubah string yang sudah ada. 

String text = "Hello";
text = text + " World";

// "Hello" tetap ada di memori
// "Hello World" adalah objek baru
// Variable text sekarang mereferensi objek baru

Kenapa String Immutable?

  • Security - String sering digunakan untuk password, username, koneksi database
  • Synchronization - Aman digunakan dalam multi-threading
  • Performance - String Pool meningkatkan efisiensi memori
  • Caching - Hash code dapat di-cache karena nilai tidak berubah

String Pool

String Pool adalah area memori khusus dalam Java Heap yang menyimpan string literal. Tujuannya adalah untuk menghemat memori dengan menggunakan kembali objek string yang identik.

Cara Kerja String Pool:

// String Literal - disimpan di String Pool
String str1 = "Hello";
String str2 = "Hello";
// str1 dan str2 mereferensi objek yang SAMA di pool

// Menggunakan new - membuat objek baru di heap
String str3 = new String("Hello");
// str3 mereferensi objek BERBEDA di heap

System.out.println(str1 == str2);  // true (referensi sama)
System.out.println(str1 == str3);  // false (referensi berbeda)
Penting: Method intern() dapat digunakan untuk memasukkan string ke pool secara manual. 
String str3 = new String("Hello").intern();
System.out.println(str1 == str3);  // true (sekarang sama)

Perbandingan String

Ada dua cara utama untuk membandingkan string di Java, dan keduanya memiliki fungsi yang berbeda.

equals()

Fungsi: Membandingkan isi/konten string

Return: boolean (true/false)

Case-Sensitive: Ya

Rekomendasi: Gunakan untuk membandingkan teks

str1.equals(str2)

Operator ==

Fungsi: Membandingkan referensi objek

Return: boolean (true/false)

Alamat Memori: Ya

Rekomendasi: Hindari untuk string

str1 == str2 
String s1 = "Hello";
String s2 = "Hello";
String s3 = new String("Hello");

// Perbandingan dengan equals()
System.out.println(s1.equals(s2));     // true (isi sama)
System.out.println(s1.equals(s3));     // true (isi sama)

// Perbandingan dengan ==
System.out.println(s1 == s2);          // true (referensi sama, dari pool)
System.out.println(s1 == s3);          // false (referensi berbeda)

// Case-insensitive comparison
System.out.println(s1.equalsIgnoreCase("hello"));  // true

Manipulasi String

Java menyediakan berbagai method untuk memanipulasi string. Berikut adalah method-method yang paling sering digunakan.

Method Umum String:

String text = "  Hello World  ";

// Panjang string
int len = text.length();              // 15

// Mengambil karakter
char ch = text.charAt(2);             // 'H'

// Substring
String sub1 = text.substring(7);      // "World  "
String sub2 = text.substring(2, 7);   // "Hello"

// Case conversion
String upper = text.toUpperCase();    // "  HELLO WORLD  "
String lower = text.toLowerCase();    // "  hello world  "

// Menghapus spasi di awal dan akhir
String trimmed = text.trim();         // "Hello World"

// Replace
String replaced = text.replace("World", "Java");  // "  Hello Java  "

// Split
String[] words = "Java,Python,C++".split(",");    // ["Java", "Python", "C++"]

// Concat
String concat = "Hello".concat(" World");         // "Hello World"
String plus = "Hello" + " World";                 // "Hello World"

// Contains
boolean has = text.contains("World"); // true

// Starts/Ends with
boolean starts = text.startsWith("  Hello");  // true
boolean ends = text.endsWith("World  ");      // true

// Index of
int index = text.indexOf("World");    // 8
int lastIndex = text.lastIndexOf("o"); // 10

StringBuilder dan StringBuffer

Karena String bersifat immutable, manipulasi string yang sering dapat menyebabkan pembuatan banyak objek baru. Java menyediakan StringBuilder dan StringBuffer sebagai alternatif yang mutable.

StringBuilder

  • Mutable - Dapat diubah tanpa membuat objek baru
  • Non-synchronized - Lebih cepat, tidak thread-safe
  • Rekomendasi - Gunakan untuk single-threaded environment
StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");        // Memodifikasi objek yang sama
sb.insert(5, " Java");      // "Hello Java World"
sb.delete(5, 10);           // "Hello World"
sb.reverse();               // "dlroW olleH"
String result = sb.toString();  // Convert ke String

StringBuffer

  • Mutable - Dapat diubah tanpa membuat objek baru
  • Synchronized - Thread-safe, lebih lambat
  • Rekomendasi - Gunakan untuk multi-threaded environment
StringBuffer sbf = new StringBuffer("Hello");
sbf.append(" World");       // Thread-safe operation
sbf.insert(5, " Java");
String result = sbf.toString();

Perbandingan Performance:

// String - Membuat 10000 objek baru (LAMBAT)
String str = "";
for (int i = 0; i < 10000; i++) {
    str = str + i;  // Setiap iterasi membuat objek baru
}

// StringBuilder - Memodifikasi 1 objek (CEPAT)
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++) {
    sb.append(i);   // Memodifikasi objek yang sama
}
String result = sb.toString();

Pertanyaan 1: Apa yang dimaksud dengan String bersifat immutable?

A. String tidak bisa dibuat
B. String tidak dapat diubah, setiap modifikasi membuat objek baru
C. String hanya bisa berisi angka
D. String tidak bisa di-print

Pertanyaan 2: Kapan sebaiknya menggunakan StringBuilder?

A. Ketika string tidak pernah diubah
B. Ketika banyak manipulasi string dalam single-threaded environment
C. Ketika perlu thread-safe operation
D. StringBuilder tidak berguna

Pertanyaan 3: Apa perbedaan antara equals() dan == untuk String?

A. Tidak ada perbedaan
B. equals() membandingkan isi, == membandingkan referensi
C. equals() lebih lambat dari ==
D. == tidak bisa digunakan untuk String

  • String Methods: length(), charAt(), substring(), trim(), replace(), split()
  • StringBuilder: Mutable, cepat, tidak thread-safe
  • StringBuffer: Mutable, thread-safe, lebih lambat dari StringBuilder
