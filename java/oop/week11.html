Week 11: Exceptions dan Assertions

Memahami penanganan kesalahan program dan validasi asumsi kode

Apa itu Exception?

Exception adalah peristiwa yang mengganggu aliran normal program, yang mewakili kesalahan atau situasi yang tidak terduga. Exception memungkinkan program menangani error dengan elegan tanpa crash.

Kenapa Exception Penting? Tanpa exception handling, program akan berhenti mendadak (crash) saat terjadi error. Dengan exception, kita dapat menangkap error, memberikan pesan yang jelas, dan program tetap berjalan.

Contoh Situasi Exception:

Arithmetic Error

Pembagian dengan nol

Null Pointer

Akses objek yang null

Array Index

Index array di luar batas

File Not Found

File yang dicari tidak ada

Visualisasi: Program Tanpa vs Dengan Exception Handling

Tanpa Exception Handling 

int result = 10 / 0;
System.out.println("Program continues");
// Program CRASH!
// Baris ini tidak akan pernah dijalankan
Output: 
Exception in thread "main"
java.lang.ArithmeticException: / by zero
Program TERMINATED

Dengan Exception Handling 

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e.getMessage());
}
System.out.println("Program continues");
Output: 
Error: / by zero
Program continues
Program SUCCESS

Hirarki Exception

Semua exception dan error di Java merupakan turunan dari kelas Throwable. Hirarki ini membagi exception menjadi beberapa kategori.

Throwable
├── Error (Unchecked)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
│
└── Exception
    ├── Checked Exceptions
    │   ├── IOException
    │   ├── SQLException
    │   ├── FileNotFoundException
    │   └── ClassNotFoundException
    │
    └── RuntimeException (Unchecked)
        ├── NullPointerException
        ├── ArithmeticException
        ├── ArrayIndexOutOfBoundsException
        ├── IllegalArgumentException
        └── NumberFormatException

Jenis-Jenis Exception

Checked Exceptions

Definisi: Exception yang diperiksa saat kompilasi dan wajib ditangani atau dideklarasikan dengan throws.

  • Kapan terjadi: Saat program berinteraksi dengan sumber daya eksternal (file, network, database)
  • Wajib ditangani: Ya, compiler memaksa programmer menangani
  • Contoh: IOException, SQLException, FileNotFoundException
// Harus ditangani atau deklarasi throws
public void readFile() throws IOException {
    FileReader file = new FileReader("data.txt");
}

// Atau tangani dengan try-catch
public void readFile() {
    try {
        FileReader file = new FileReader("data.txt");
    } catch (IOException e) {
        System.out.println("File tidak ditemukan");
    }
}

Unchecked Exceptions (RuntimeException)

Definisi: Exception akibat logika pemrograman yang tidak wajib ditangani secara eksplisit.

  • Kapan terjadi: Kesalahan logika program (bug)
  • Wajib ditangani: Tidak, tapi sebaiknya dicegah dengan validasi
  • Contoh: NullPointerException, ArithmeticException, ArrayIndexOutOfBoundsException
// Tidak wajib try-catch, tapi bisa crash
String text = null;
int length = text.length();

// Lebih baik: validasi dulu
if (text != null) {
    int length = text.length();
}

Error

Definisi: Masalah serius yang umumnya tidak dapat dipulihkan oleh program.

  • Kapan terjadi: Masalah pada JVM atau sistem
  • Wajib ditangani: Tidak, dan sebaiknya tidak ditangani
  • Contoh: OutOfMemoryError, StackOverflowError
// Jangan tangani Error, biarkan program terminate
public void recursiveMethod() {
    recursiveMethod();
}
// StackOverflowError - tidak bisa dipulihkan

Try-Catch-Finally

Struktur try-catch-finally adalah mekanisme utama untuk menangani exception di Java.

Struktur Dasar:

try {
    // Kode yang mungkin throw exception
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Tangani exception spesifik
    System.out.println("Tidak bisa bagi dengan nol");
} catch (Exception e) {
    // Tangani exception umum
    System.out.println("Terjadi kesalahan: " + e.getMessage());
} finally {
    // Selalu dieksekusi, baik ada exception atau tidak
    System.out.println("Blok finally dijalankan");
}

Multiple Catch Blocks:

try {
    String text = null;
    int[] arr = {1, 2, 3};
    
    System.out.println(text.length());
    System.out.println(arr[10]);
} catch (NullPointerException e) {
    System.out.println("Objek null!");
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("Index array melebihi batas!");
} catch (Exception e) {
    System.out.println("Exception lain: " + e.getMessage());
}

Multi-Catch (Java 7+):

try {
    // Some code
} catch (IOException | SQLException e) {
    // Handle multiple exceptions dengan satu catch
    System.out.println("IO atau SQL Error: " + e.getMessage());
}
Penting tentang Finally:
  • Finally block selalu dieksekusi, bahkan jika ada return di try atau catch
  • Digunakan untuk cleanup: menutup file, koneksi database, resource lainnya
  • Finally bersifat optional (boleh tidak ada)

Throws dan Throw

Throws - Deklarasi Exception:

throws digunakan pada signature method untuk mendeklarasikan bahwa method tersebut dapat melempar exception.

public void readFile(String filename) throws IOException {
    FileReader file = new FileReader(filename);
    // Method ini tidak handle exception, delegasikan ke caller
}

// Caller harus handle
public void processFile() {
    try {
        readFile("data.txt");
    } catch (IOException e) {
        System.out.println("Error: " + e.getMessage());
    }
}

Throw - Melempar Exception:

throw digunakan untuk melempar exception secara eksplisit.

public void setAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("Umur tidak boleh negatif");
    }
    this.age = age;
}

// Custom Exception
public void withdraw(double amount) throws InsufficientFundsException {
    if (amount > balance) {
        throw new InsufficientFundsException("Saldo tidak cukup");
    }
    balance -= amount;
}
Perbedaan Throws vs Throw:
  • throws - Deklarasi di method signature
  • throw - Melempar exception instance

Custom Exception

Anda dapat membuat exception sendiri dengan meng-extend class Exception atau RuntimeException.

// Custom Checked Exception
public class InsufficientFundsException extends Exception {
    private double amount;
    
    public InsufficientFundsException(double amount) {
        super("Saldo tidak cukup. Kekurangan: " + amount);
        this.amount = amount;
    }
    
    public double getAmount() {
        return amount;
    }
}

// Custom Unchecked Exception
public class InvalidEmailException extends RuntimeException {
    public InvalidEmailException(String email) {
        super("Email tidak valid: " + email);
    }
}

// Penggunaan
public void transfer(double amount) throws InsufficientFundsException {
    if (amount > balance) {
        throw new InsufficientFundsException(amount - balance);
    }
    balance -= amount;
}

Assertions

Assertions adalah pernyataan untuk menguji asumsi selama pengembangan (debugging). Jika kondisi assertion bernilai false, program melempar AssertionError.

Syntax Assertion:

// Syntax 1: assert condition
assert age >= 0;

// Syntax 2: assert condition : message
assert age >= 0 : "Umur tidak boleh negatif, nilai: " + age;

// Contoh penggunaan
public void setAge(int age) {
    assert age >= 0 && age <= 150 : "Invalid age: " + age;
    this.age = age;
}

Karakteristik Assertions

  • Default: Assertions dinonaktifkan (disabled)
  • Aktivasi: Gunakan flag -ea atau -enableassertions saat run
  • Tujuan: Untuk debugging dan development, bukan production
  • Performance: Tidak impact performance saat disabled

Cara Mengaktifkan Assertions:

// Enable semua assertions
java -ea MyProgram

// Enable assertions untuk package tertentu
java -ea:com.myapp... MyProgram

// Enable assertions untuk class tertentu
java -ea:com.myapp.MyClass MyProgram

// Disable assertions (default)
java MyProgram

Kapan Menggunakan Assertions:

Gunakan Assertions Untuk:

  • Internal invariants
  • Control flow invariants
  • Preconditions (private methods)
  • Postconditions
  • Development testing

Jangan Gunakan Assertions Untuk:

  • Validasi input user
  • Public method preconditions
  • Operasi dengan side effects
  • Production error handling
  • Command-line arguments
// BENAR: Internal invariant
private void setAge(int age) {
    assert age >= 0 : "Age must be non-negative";
    this.age = age;
}

// SALAH: Jangan untuk public method
public void setUserAge(int age) {
    // JANGAN gunakan assert untuk validasi user input
    // assert age >= 0;  // SALAH!
    
    // Gunakan exception untuk validasi user input
    if (age < 0) {
        throw new IllegalArgumentException("Age must be non-negative");
    }
    this.age = age;
}

  • Exception: Peristiwa yang mengganggu aliran normal program
  • Hirarki: Throwable → Error & Exception → Checked & Unchecked
  • Checked Exception: Wajib ditangani, diperiksa saat kompilasi
  • Unchecked Exception: RuntimeException, tidak wajib ditangani
  • Try-Catch-Finally: Mekanisme penanganan exception
  • Throws: Deklarasi method bisa throw exception
  • Throw: Melempar exception secara eksplisit
