Week 14: Multithreading

Menjalankan beberapa alur eksekusi secara bersamaan untuk meningkatkan kinerja aplikasi

Definisi Thread

Thread adalah proses ringan dan unit eksekusi terkecil dalam program. Setiap thread memiliki alur eksekusi independen dalam satu proses.

Proses vs Thread: Proses adalah instansi program yang sedang berjalan dan dapat memiliki banyak thread. Thread berbagi memori dalam satu proses.

Single-threaded

Eksekusi sequential, satu tugas selesai baru tugas berikutnya

Contoh: Program sederhana tanpa concurrency

Multi-threaded

Eksekusi paralel, beberapa tugas berjalan bersamaan

Contoh: Server web, aplikasi GUI

Pembuatan Thread

Ada dua cara utama membuat thread di Java:

1. Extends Thread Class

Cara: Buat kelas yang extends Thread dan override method run()

class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(Thread.currentThread().getName() + " - " + i);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

MyThread t1 = new MyThread();
MyThread t2 = new MyThread();
t1.start();
t2.start();

2. Implements Runnable Interface

Cara: Implementasikan interface Runnable dan berikan ke Thread constructor

class MyTask implements Runnable {
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(Thread.currentThread().getName() + " - " + i);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

Thread t1 = new Thread(new MyTask());
Thread t2 = new Thread(new MyTask());
t1.start();
t2.start();
Best Practice: Gunakan Runnable interface karena lebih fleksibel (Java tidak mendukung multiple inheritance).

Siklus Hidup Thread

Thread memiliki berbagai status selama masa hidupnya:

1
NEW

Thread baru dibuat tetapi belum dimulai

2
RUNNABLE

Thread siap untuk dieksekusi, menunggu CPU scheduler

3
RUNNING

Thread sedang dieksekusi oleh CPU

4
BLOCKED / WAITING

Thread menunggu resource atau thread lain selesai

5
TERMINATED

Thread telah selesai eksekusi atau dihentikan

Metode Penting Thread

class ThreadDemo extends Thread {
    @Override
    public void run() {
        System.out.println("Thread running: " + getName());
    }
}

Thread t1 = new ThreadDemo();
Thread t2 = new ThreadDemo();

t1.start();

try {
    Thread.sleep(2000);
} catch (InterruptedException e) {
    e.printStackTrace();
}

t2.start();

try {
    t1.join();
    t2.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}

System.out.println("All threads completed");

Method-method Utama:

Method Deskripsi
start() Memulai eksekusi thread, memanggil method run()
sleep(ms) Menghentikan sementara thread selama waktu tertentu (milliseconds)
join() Menunggu thread lain selesai sebelum melanjutkan
getName() Mendapatkan nama thread
setName() Mengatur nama thread
getPriority() Mendapatkan prioritas thread (1-10)

Sinkronisasi

Sinkronisasi diperlukan ketika multiple threads mengakses shared resources untuk mencegah race condition dan data corruption.

Race Condition: Kondisi dimana output program bergantung pada urutan eksekusi thread yang tidak dapat diprediksi.

Synchronized Method

class Counter {
    private int count = 0;
    
    public synchronized void increment() {
        count++;
    }
    
    public synchronized int getCount() {
        return count;
    }
}

Counter counter = new Counter();

Thread t1 = new Thread(() -> {
    for (int i = 0; i < 1000; i++) {
        counter.increment();
    }
});

Thread t2 = new Thread(() -> {
    for (int i = 0; i < 1000; i++) {
        counter.increment();
    }
});

t1.start();
t2.start();

try {
    t1.join();
    t2.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}

System.out.println("Count: " + counter.getCount());

Synchronized Block

class BankAccount {
    private int balance = 0;
    private final Object lock = new Object();
    
    public void deposit(int amount) {
        synchronized (lock) {
            balance += amount;
            System.out.println("Deposited: " + amount + ", Balance: " + balance);
        }
    }
    
    public void withdraw(int amount) {
        synchronized (lock) {
            if (balance >= amount) {
                balance -= amount;
                System.out.println("Withdrew: " + amount + ", Balance: " + balance);
            }
        }
    }
}
Kapan menggunakan synchronized: Gunakan synchronized hanya pada critical section untuk menghindari bottleneck performa.

Ringkasan Week 14

  • Thread: Unit eksekusi terkecil yang berjalan dalam satu proses
  • Pembuatan Thread: Extends Thread class atau implements Runnable interface
  • Lifecycle: NEW → RUNNABLE → RUNNING → BLOCKED/WAITING → TERMINATED
  • start(): Memulai eksekusi thread
  • sleep(): Menghentikan sementara eksekusi thread
  • join(): Menunggu thread lain selesai
  • synchronized: Mencegah race condition dengan membatasi akses ke shared resources
  • Best Practice: Gunakan Runnable, synchronize critical sections saja
