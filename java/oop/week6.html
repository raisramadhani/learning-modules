Week 6: Polymorphism

Memahami polimorfisme, method overloading, overriding, dan dynamic binding

Apa itu Polymorphism?

Polymorphism berarti "banyak bentuk". Dalam OOP, polymorphism memungkinkan objek diperlakukan sebagai instance dari superclass-nya, meskipun sebenarnya adalah instance dari subclass.

Runtime Polymorphism (Dynamic Binding)

Dicapai melalui method overriding dan upcasting. Method yang dipanggil ditentukan saat runtime berdasarkan tipe objek yang sebenarnya.

class Shape {
    void draw() {
        System.out.println("Drawing a shape");
    }
}

class Circle extends Shape {
    @Override
    void draw() {
        System.out.println("Drawing a circle ‚≠ï");
    }
}

class Square extends Shape {
    @Override
    void draw() {
        System.out.println("Drawing a square ‚¨õ");
    }
}

// Runtime Polymorphism - Upcasting
Shape shape1 = new Circle();  // Upcasting
Shape shape2 = new Square();  // Upcasting

shape1.draw();  // Output: Drawing a circle ‚≠ï
shape2.draw();  // Output: Drawing a square ‚¨õ

// Method yang dipanggil ditentukan saat runtime!

Compile-time Polymorphism (Static Binding)

Dicapai melalui method overloading. Beberapa method dengan nama sama tapi parameter berbeda (jumlah atau tipe).

Syarat Method Overloading:

  • ‚úì Nama method harus sama
  • ‚úì Parameter harus berbeda (jumlah atau tipe)
  • ‚úì Return type boleh sama atau berbeda
  • ‚úì Dalam class yang sama atau inheritance
class Calculator {
    // Overloading dengan jumlah parameter berbeda
    int add(int a, int b) {
        return a + b;
    }
    
    int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Overloading dengan tipe parameter berbeda
    double add(double a, double b) {
        return a + b;
    }
    
    String add(String a, String b) {
        return a + b;
    }
}

// Penggunaan
Calculator calc = new Calculator();
System.out.println(calc.add(5, 3));           // Output: 8
System.out.println(calc.add(5, 3, 2));        // Output: 10
System.out.println(calc.add(5.5, 3.2));       // Output: 8.7
System.out.println(calc.add("Hello", " World")); // Output: Hello World

‚ùå Operator Overloading

‚ö†Ô∏è Catatan Penting:

Java TIDAK mendukung operator overloading (berbeda dengan C++). Anda tidak bisa mengubah arti operator seperti +, -, * untuk class buatan Anda.

Pengecualian: String menggunakan + untuk concatenation, tapi ini built-in dari Java.

// Di C++ bisa (tapi di Java TIDAK BISA):
// Complex operator+(Complex c1, Complex c2) { ... }

// Di Java, harus menggunakan method biasa:
class Complex {
    int real, imag;
    
    Complex add(Complex other) {  // Method biasa, bukan operator overloading
        Complex result = new Complex();
        result.real = this.real + other.real;
        result.imag = this.imag + other.imag;
        return result;
    }
}

Pertanyaan: Apa perbedaan utama antara Overloading dan Overriding?

A. Tidak ada perbedaan
B. Overloading: nama sama parameter beda. Overriding: subclass ubah method parent
C. Overloading hanya untuk static method
D. Overriding tidak bisa mengubah implementasi
