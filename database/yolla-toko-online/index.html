Aturan Syntax SQL

Sebelum mengetik, pahami 3 aturan emas penulisan SQL:

  • Titik Koma (;): Wajib ada di akhir setiap perintah sebagai penutup instruksi.
  • Tanda Kutip ('...'): Wajib digunakan saat mengisi data teks (String) atau tanggal.
  • Huruf Besar: Perintah SQL (seperti CREATE, TABLE) disarankan menggunakan HURUF KAPITAL agar mudah dibedakan dengan nama tabel/kolom.
  • Tanda strip (-): Merupakan comment/komen dalam SQL.

Initial Setup

Materi: Wadah Database

Sebelum membuat tabel, kita harus menyiapkan wadahnya.

  • CREATE DATABASE: Perintah untuk membangun gedung data baru.
  • USE: Perintah krusial untuk memberitahu sistem bahwa kita "masuk" dan bekerja di dalam database tersebut.
SQL Editor Ketik Manual 
CREATE DATABASE db_tokoonline;

-- Jangan lupa jalankan baris ini agar tabel tidak nyasar ke database lain
USE db_tokoonline;
2

Master Data & Tipe Data

Materi: Tipe Data & Constraints

Tipe Data String:
  • VARCHAR(n): Teks pendek variabel (Nama, Email). Hemat memori.
  • TEXT: Teks panjang tanpa batas spesifik (Alamat, Deskripsi).
Constraints (Batasan):
  • PRIMARY KEY: Kunci utama, data tidak boleh kembar.
  • AUTO_INCREMENT: Nomor urut otomatis (1, 2, 3..).
  • ENUM: Pilihan ganda terbatas (Radio Button).

Tugas: Buat tabel Pengguna dan Kategori.

Table: Pengguna & Kategori Ketik Manual 
-- 1. Tabel Pengguna (Menampung Admin & Pembeli)
CREATE TABLE pengguna (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama_lengkap VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL, -- Email tidak boleh sama
    password VARCHAR(255) NOT NULL,
    no_hp VARCHAR(20),
    alamat_utama TEXT, -- Menggunakan TEXT karena alamat bisa panjang
    role ENUM('admin', 'pelanggan') DEFAULT 'pelanggan',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. Tabel Kategori
CREATE TABLE kategori (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama_kategori VARCHAR(50) NOT NULL,
    slug VARCHAR(50)
);
3

Relasi Antar Tabel (Foreign Key)

Materi: Foreign Key Concept

Agar database saling terhubung, kita menggunakan Foreign Key (FK). FK adalah kolom "tamu" dari tabel lain.

  • Struktur: FOREIGN KEY (kolom_ini) REFERENCES tabel_lain(id).
  • ON DELETE CASCADE: Fitur keamanan data. Jika data "Induk" dihapus, maka data "Anak" otomatis terhapus bersih. Contoh: Jika Produk 'Keripik' dihapus, maka Stok 'Keripik' juga hilang.
Table: Produk, Varian, Stok Ketik Manual 
-- 3. Tabel Produk (Induk)
CREATE TABLE produk (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_kategori INT NOT NULL,
    nama_produk VARCHAR(100) NOT NULL,
    deskripsi TEXT,
    foto_utama VARCHAR(255),
    -- Definisi Relasi ke Kategori
    FOREIGN KEY (id_kategori) REFERENCES kategori(id) ON DELETE CASCADE
);

-- 4. Tabel Varian (Detail Rasa/Ukuran)
CREATE TABLE varian_produk (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_produk INT NOT NULL, -- Kolom tamu (FK)
    nama_varian VARCHAR(100) NOT NULL, 
    harga DECIMAL(10, 2) NOT NULL, -- Menggunakan DECIMAL untuk akurasi uang
    berat_gram INT NOT NULL,
    tanggal_kedaluwarsa DATE, -- Format YYYY-MM-DD
    FOREIGN KEY (id_produk) REFERENCES produk(id) ON DELETE CASCADE
);

-- 5. Tabel Stok
CREATE TABLE stok (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_varian INT NOT NULL,
    jumlah INT NOT NULL DEFAULT 0,
    -- Mencatat waktu update terakhir otomatis
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (id_varian) REFERENCES varian_produk(id) ON DELETE CASCADE
);
4

Transaksi & Keuangan

Materi: Presisi Uang (DECIMAL)

Jangan pernah gunakan FLOAT atau DOUBLE untuk uang karena pembulatan koma yang tidak akurat. Gunakan DECIMAL(10, 2).

  • 10: Total digit angka (termasuk koma).
  • 2: Jumlah digit di belakang koma (sen).
Table: Keranjang & Pesanan Ketik Manual 
-- 6. Tabel Keranjang
CREATE TABLE keranjang (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pengguna INT NOT NULL,
    id_varian INT NOT NULL,
    jumlah INT NOT NULL DEFAULT 1,
    FOREIGN KEY (id_pengguna) REFERENCES pengguna(id) ON DELETE CASCADE,
    FOREIGN KEY (id_varian) REFERENCES varian_produk(id) ON DELETE CASCADE
);

-- 7. Tabel Pesanan (Header Faktur)
CREATE TABLE pesanan (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pengguna INT NOT NULL,
    tanggal_pesanan DATETIME DEFAULT CURRENT_TIMESTAMP,
    total_tagihan DECIMAL(12, 2) NOT NULL,
    status_pesanan ENUM('pending', 'dibayar', 'diproses', 'dikirim', 'selesai', 'batal') DEFAULT 'pending',
    catatan_pembeli TEXT,
    FOREIGN KEY (id_pengguna) REFERENCES pengguna(id)
);

-- 8. Tabel Detail Pesanan (Isi Faktur)
-- Mencatat harga saat beli agar history aman jika harga produk naik
CREATE TABLE detail_pesanan (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pesanan INT NOT NULL,
    id_varian INT NOT NULL,
    harga_saat_beli DECIMAL(10, 2) NOT NULL,
    jumlah_beli INT NOT NULL,
    subtotal DECIMAL(12, 2) NOT NULL,
    FOREIGN KEY (id_pesanan) REFERENCES pesanan(id) ON DELETE CASCADE,
    FOREIGN KEY (id_varian) REFERENCES varian_produk(id)
);
5

Finishing (Logistik)

Terakhir, kita buat tabel untuk menyimpan bukti transfer dan resi pengiriman.

Table: Pembayaran & Pengiriman Ketik Manual 
-- 9. Tabel Pembayaran
CREATE TABLE pembayaran (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pesanan INT NOT NULL,
    metode_pembayaran VARCHAR(50),
    tanggal_bayar DATETIME,
    bukti_bayar VARCHAR(255),
    status_pembayaran ENUM('menunggu', 'lunas', 'gagal') DEFAULT 'menunggu',
    FOREIGN KEY (id_pesanan) REFERENCES pesanan(id) ON DELETE CASCADE
);

-- 10. Tabel Pengiriman
CREATE TABLE pengiriman (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pesanan INT NOT NULL,
    kurir VARCHAR(50),
    layanan VARCHAR(50),
    no_resi VARCHAR(100),
    alamat_tujuan TEXT NOT NULL,
    ongkos_kirim DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (id_pesanan) REFERENCES pesanan(id) ON DELETE CASCADE
);
